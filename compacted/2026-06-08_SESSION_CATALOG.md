## Registration crash on iPhone Safari (ebebe722)
Pete: new person's dashboard won't load on Safari. Diagnosed from the **`client_errors`** table (kind `unhandledrejection`, iPhone Safari UA): *"cannot add postgres_changes callbacks for realtime:society_events_changes after subscribe()."* Cause: `SocietyGolfDB.subscribeToEvents` reuses the channel name `society_events_changes` and is called from 4+ sites; Supabase keys channels by name, so the 2nd caller `.on()`'d an already-subscribed channel → throw → unhandled rejection halted the dashboard load (Safari timing made the double-call land). Fix: multiplex onto ONE shared channel (`_eventChannel` + `_eventCallbacks`; repeat callers just push a callback); reset in `unsubscribeAll`. See [[realtime-channel-reuse-crash]].
