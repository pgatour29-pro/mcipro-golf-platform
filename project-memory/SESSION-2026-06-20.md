## 2. Work by area

### A. Admin — phantom "active users" (`69ef2ddb`)
- **Bug:** the User Activity panel used `user_profiles.updated_at` as "last active." A global handicap
  update bumped 9 users' `updated_at` to the same instant → all showed "Online/active today." Proven in DB.
- **Fix:** added `user_profiles.last_login_at`; `recordUserLogin()` stamps it once per session on dashboard
  entry; "true last active" = max(login, last round, last event) — never `updated_at`.
- Full writeup: FUCKUPS.md #1.

### B. Design system brief (`8fa1635d`)
- claude.ai/design produced a poor golfer dashboard; `/design-sync` doesn't apply (MciPro is a monolithic
  vanilla app, no component library). Built **Option A**: `project-memory/DESIGN-SYSTEM.md` — a tokens +
  conventions brief (real Tailwind classes from the code) that doubles as a claude.ai/design prompt.
- Key fact: MciPro has **no custom design tokens** — stock Tailwind via CDN + brand conventions.

### C. Golfer overview cubes — live info (`8f18b85d`, `0a0f0d3f`, `c9a8f2b4`, `b2fc25b6`)
- **`GolferCubeInfo` module** binds each overview cube to its live data (kept the existing cube design;
  Pete rejected a restyle). Defensive — only overwrites a pill when it has real data; falls back to the
  original label otherwise (zero regression risk).
- **Mobile fix (`0a0f0d3f`):** mobile CSS hid `.mt-2` pills (`display:none`) so the info was invisible on
  phones. Tagged live pills `.cube-live` and showed those on mobile; empty cubes stay compact.
- **Descriptions → data (`c9a8f2b4`):** removed cube subtitle descriptions, every cube shows a data line
  (with empty states); Play Golf kept its CTA.
- **Correct data source (`b2fc25b6`):** cubes initially read tee-time *bookings* + the new-events badge →
  showed "No upcoming." Pete's rounds are **society-event registrations**. `loadEvents()` now queries
  `event_registrations → society_events`: Tee Time = next event, Society = "N upcoming", Caddy = "Caddy #".
- Sources also wired: rounds count + best (rounds table), live weather (Open-Meteo).

### D. Today's Tee Time box — rollover (`10c52fb1`)
- Per Pete: show the **current event**, and once its time passes roll to the **next upcoming**.
- `renderTeeBox()` fills the box from the same event source (course, tee + departure, caddy, weather);
  title switches "Today's Tee Time" ⇄ "Next Round"; ~5h grace keeps the round visible while in progress.

### E. Caddies + back button (`7cbdab61`, `26d9c133`; `ecea9347`→`331236bb` reverted)
- **Back button (`7cbdab61`):** box links ("Schedule →", "Book Caddy →", "View All →") used
  `TabManager.showTab` which doesn't push `NavHistory` → the universal back button stayed hidden. Switched
  to `showGolferTab` (pushes history).
- **My Caddies demo (`26d9c133`):** `caddy_profiles` (128) is **seed/demo** data. Per Pete, demo caddies

  + `event_registrations.caddy_numbers`. Verified: Pete = 73 real caddies, zero demo. Also resolved the
  "Siam caddies under Pattaya CC" complaint (no demo left in the module to mis-bucket).
